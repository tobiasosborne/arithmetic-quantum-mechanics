## Findings

- **Major - AQM-08 has a sign mismatch in the claimed induced logical pairing.**
  The shard fixes the symplectic convention as
  `omega((x,z),(x',z')) = z*x' - x*z'` at
  `report/sections/08_symplectic_css_bridge.tex:55-58`, and it explicitly
  computes the mixed CSS pairing as `omega((x,0),(0,z))=-x*z` at
  `report/sections/08_symplectic_css_bridge.tex:89-92`. But the logical
  Lagrangian paragraph later says the induced pairing
  `H^1 x H_1 -> F_p` is `([x],[z]) -> x(z)` at
  `report/sections/08_symplectic_css_bridge.tex:222-226`. With the recorded
  convention, the inherited symplectic form on the ordered pair
  `((x,0),(0,z))` is `-x(z)`, not `+x(z)`. This is invisible over `F_2` but
  matters for the `p=3,5` setting claimed in the same shard's validation
  paragraph.

- **Major - AQM-08 appears to swap the logical X/Z polarization names.**
  The shard states that the first component is the `X` exponent and the second
  is the `Z` exponent at `report/sections/08_symplectic_css_bridge.tex:42-43`.
  It then defines
  `Lambda_Z=Z^1 op B_1` and `Lambda_X=B^1 op Z_1` at
  `report/sections/08_symplectic_css_bridge.tex:212-220`. Since
  `Lambda_Z/L` is represented by the first component `H^1`, this is the
  `X`-label half under the shard's own convention; `Lambda_X/L`, represented
  by `H_1` in the second component, is the `Z`-label half. This also conflicts
  with the explicit Steane convention
  `Xbar=(u,0)`, `Zbar=(0,u)` in `CONVENTIONS.md:312-319`.

- **Major - AQM-08 states a general stabilizer dimension claim from exponent
  labels without recording the necessary phase/eigencharacter caveat.**
  `report/sections/08_symplectic_css_bridge.tex:60-68` says an isotropic
  exponent-label subspace `L` gives commuting independent stabilizers with
  joint eigenspace dimension `p^(n-r)`. For the later CSS construction this is
  repaired by canonical CSS lifts, but as written the general statement does
  not say which signed/Hermitian Pauli representatives or stabilizer character
  are being used. This is convention-sensitive, especially for binary
  non-CSS labels such as labels with both `X` and `Z` components. The later
  signed-Pauli convention in `CONVENTIONS.md:347-358` shows that signs cannot
  be silently suppressed in arbitrary stabilizer statements.

- **Minor - AQM-09 source anchors are less precise than the local citation
  discipline asks for.** `report/sections/09_symplectic_supercharge_dictionary.tex:17-21`
  cites only `Thesis.tex`, `n9.tex`, and `HomologicalErrorCorrection6.tex`
  with line ranges, while AQM-08 gives full local paths. The manifests do
  register the full local paths, so the references are not broken, but this
  shard should cite the local paths directly for consistency with the rule that
  claims point to local paths plus locators.

- **Question - Should AQM-09 narrow the wording of its validation claim?**
  `report/sections/09_symplectic_supercharge_dictionary.tex:272-278` says the
  script validates the dictionary and records `Q^2` and anticommutator
  certificates. The local producer records those certificates as algebraic
  booleans derived from CSS isotropy, not from Hilbert-matrix or projector
  construction checks. The text does say no Hilbert matrices are built, so this
  may be intentional, but the report should make clear that the run is a
  finite example/dimension certificate and the operator identities are carried
  by the preceding formal derivation.

## Cross-Shard Risks

- The sign issue in AQM-08 can propagate to later odd-prime qudit shards if
  they use `H^1 x H_1 -> F_p` as the symplectic pairing rather than as an
  unsigned evaluation pairing.
- The `Lambda_X`/`Lambda_Z` name swap risks confusing AQM-10 and later Steane
  shards, where the repository convention explicitly treats `(u,0)` as
  `Xbar` and `(0,u)` as `Zbar`.
- The phase caveat in the general stabilizer paragraph should be aligned with
  the signed Pauli/Koszul convention used in later Clifford and
  ghost-Gaussian shards.

## Suggested Actions

- In AQM-08, either change the logical pairing to `([x],[z]) -> -x(z)` for the
  ordered `H^1 x H_1` pairing, or state the pairing in the reversed order
  `H_1 x H^1 -> F_p`.
- Rename the displayed logical polarizations, or add a sentence explaining
  that the names refer to a computational-basis convention rather than to the
  Pauli-label component. The safer local fix is likely
  `Lambda_X=Z^1 op B_1` and `Lambda_Z=B^1 op Z_1`.
- Add a caveat to the general stabilizer paragraph that exponent labels alone
  determine the symplectic centralizer, while an actual stabilizer code also
  needs compatible phases/eigencharacter; then note that the CSS canonical
  lifts used in the rest of the shard supply that choice.
- Replace the bare AQM-09 source filenames with the full local paths already
  used in AQM-08.
- Clarify that the AQM-09 validation run is a representative exact algebraic
  certificate for CSS matrices, while the proof text supplies the general
  `Q^2` and anticommutator identities.
