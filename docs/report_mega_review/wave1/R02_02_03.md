## Findings

- **Major:** `AQM-02` has stale source provenance for the Cahill--Glauber
  displacement source. The lab log says the Grassmann displacement shards use
  "Cahill--Glauber's Physical Review A source converted with `marker`"
  (`report/sections/02_lab_log.tex:66`-`69`). The current local manifest says
  the recorded APS PDF/marker chain could not be restored, those marker
  artifacts are not active report locators, and the active source chain for
  `AQM-20`/`AQM-21` was migrated to the official arXiv e-print source
  (`references/canonical_fields/SOURCES.md:71`-`78`). This is a traceability
  defect: a reader following the lab log is sent to a non-active, missing
  source chain.

- **Major:** `AQM-02` is no longer a reliable chronological lab log for the
  report. Its last entry is `2026-05-26 Minimal Fermion Catalogue`
  (`report/sections/02_lab_log.tex:272`-`279`), but the master includes many
  later shards after `AQM-53`, from `AQM-54` through `AQM-91`
  (`report.tex:85`-`122`). The catalog likewise records substantial later
  derivative/QSA/finite-ring/torus shards (`report/SHARD_CATALOG.md:487`-`820`).
  At minimum, the finite-ring database run is registered in `INDEX.md`
  (`INDEX.md:23`) and its fast producers are in `scripts/run_all.jl`
  (`scripts/run_all.jl:20`-`53`), but none of that appears in the lab log.
  Either the shard needs entries through the current report state or it should
  explicitly declare the date/shard boundary it covers.

- **Major:** `AQM-02` makes generated-data claims without the producer/run
  pointers required by `AQM-03`. The reproducibility map says every report
  claim using generated data must point back to the producer and run bundle
  (`report/sections/03_reproducibility_map.tex:18`-`21`). The lab log asserts
  that the Steane run checks standard Clifford generators
  (`report/sections/02_lab_log.tex:28`-`29`), that an arithmetic-field run
  checks finite-set/vector-space/finite-variety examples
  (`report/sections/02_lab_log.tex:37`-`38`), and that the projective-line run
  checks radicals, kernels, germs, and residue tables
  (`report/sections/02_lab_log.tex:49`-`53`) without naming the concrete
  `runs/...` bundle, script, or CSV. The information exists in `INDEX.md`
  (`INDEX.md:47`-`49`, `INDEX.md:69`-`76`), but this report shard does not
  carry the required trace.

- **Question:** Should `AQM-03` distinguish scientific evidence producers from
  infrastructure/smoke producers in the validation command block? It labels
  `scripts/run_all.jl --fast` as part of the "checked local evidence layer"
  (`report/sections/03_reproducibility_map.tex:23`-`30`), but the current fast
  driver also runs finite-ring database build/audit/export entries
  (`scripts/run_all.jl:20`-`53`) whose `INDEX.md` status is report-pending and
  schema-only/in-memory rather than a populated audited database
  (`INDEX.md:23`, `INDEX.md:50`-`52`). This may be fine, but the map should make
  the status boundary explicit so `--fast` is not read as certifying every
  registered output as checked scientific evidence.

## Cross-Shard Risks

- Source-integrity audits can supersede lab-log provenance without the lab log
  being updated, as seen with the Cahill--Glauber marker-to-arXiv migration.
- Summary claims in the early lab-log shard are easy to read as evidence
  claims, but many do not include the run/script/source paths demanded by the
  reproducibility map.
- `run_all.jl --fast` now mixes established mathematical checks with
  infrastructure/database smoke checks; downstream report text needs stable
  status labels for that distinction.

## Suggested Actions

- Replace the `marker` provenance in `AQM-02` with the active
  `references/canonical_fields/cahill_glauber_physics_9808029v1_source/...`
  arXiv locator, or add an explicit note that the marker chain is historical
  and source-integrity blocked.
- Bring `AQM-02` current through the existing report include order, or add a
  clear statement that the chronological log currently stops at `AQM-53`.
- Add concrete `runs/...`, script, and CSV references to lab-log sentences that
  summarize generated-data checks.
- In `AQM-03`, add a short status note or table separating mathematical
  evidence runs, report-pending infrastructure runs, and schema-only/local
  smoke artifacts.
