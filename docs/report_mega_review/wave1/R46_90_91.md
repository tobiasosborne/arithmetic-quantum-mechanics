## Findings

- Severity: Major
  Location: `report/sections/91_haah_laurent_torus_quant2.tex:162`
  Issue:
  The finite-periodic exactness discussion can be read as asserting
  `ker epsilon_square = im sigma_square` for the finite quotient `R_G`.
  Evidence:
  Lines 162--184 define `epsilon_square` over `P_G` and `R_G`, define
  `C_comm(G)=ker epsilon_square`, and then state "The strong toric exactness
  statement is `ker epsilon_square=im sigma_square`", with only a caveat that
  finite quotients can create global logical labels. The local Haah thesis
  locator in `references/lattice_stabilizer_codes/SOURCES.md:42` supports the
  infinite Laurent-ring toric-code exactness statement at
  `alg-theory.tex:1515-1567`; in that source, the displayed exactness is for
  the translation-invariant Laurent module before imposing a finite periodic
  quotient. In the report's own finite-periodic language, the quotient
  `ker(sigma^dagger lambda)/im(sigma)` is later identified as the place to
  record finite-periodic logical labels (`91_haah_laurent_torus_quant2.tex:219-229`),
  so equality on `R_G` is at least not established and is generally the point
  under test.
  Suggested action:
  Rephrase the equality as Haah's source-local infinite-lattice exactness
  statement over the Laurent translation ring, then state separately that on
  `R_G` the checked object is the quotient
  `ker epsilon_square / im sigma_square`, which may be nonzero because of
  periodic logical labels. Avoid presenting equality over `R_G` unless a local
  finite-periodic derivation or run is added.

- Severity: Major
  Location: `report/sections/91_haah_laurent_torus_quant2.tex:77`
  Issue:
  The finite bridge mixes a generic prime-qudit translation ring over
  `F_p` with the qubit edge-label module over `F_2` without fixing `p=2` at
  the point where the modules are identified.
  Evidence:
  Lines 77--83 define
  `R_G = F_p[X^{\pm1},Y^{\pm1}]/(X^{N_x}-1,Y^{N_y}-1)` and describe it as the
  periodic label module. Lines 92--102 then set `P_G=R_G^4` and identify it
  with
  `F_2^{G x {x,y}}_q direct_sum F_2^{G x {x,y}}_p`.
  This identification is only literal when the Haah label field is `F_2`.
  The source manifest distinguishes prime-qudit sources
  (`references/lattice_stabilizer_codes/SOURCES.md:87-99`) from the qubit
  toric-code matrix locator in the thesis (`SOURCES.md:31-43`), and AQM-90's
  Cayley-square edge layer explicitly puts qubit labels on `S_1`
  (`90_algebraic_torus_quant2.tex:193-220`).
  Suggested action:
  In the AQM-91 bridge section, either specialize to `p=2` before identifying
  `R_G^4` with AQM-90's qubit edge module, or split the text into a generic
  prime-qudit translation-ring paragraph and a separate qubit toric-square
  paragraph.

- Severity: Minor
  Location: `report/sections/91_haah_laurent_torus_quant2.tex:162`
  Issue:
  The shard's excitation-map notation is locally sensible, but it conflicts
  with convention `(az)` as currently written.
  Evidence:
  AQM-91 defines
  `epsilon_square = sigma_square^\dagger lambda_2 : P_G -> R_G^2`
  at lines 162--165 and then uses
  `sigma_square^\dagger lambda_2 sigma_square=0` as the generator-isotropy
  equation at lines 175--179. That matches the Haah source locator in
  `references/lattice_stabilizer_codes/SOURCES.md:28` and the local source
  `alg-code-hamiltonian-v6.tex:532-544`. However, `CONVENTIONS.md` convention
  `(az)` states "The excitation map is
  `epsilon_square = sigma_square^dagger lambda sigma_square`", which is the
  commutation-on-generators expression, not the map from the Pauli module to
  the excitation module.
  Suggested action:
  Keep AQM-91's distinction between `epsilon = sigma^dagger lambda` and
  `sigma^dagger lambda sigma = 0`, and update convention `(az)` in a later
  non-review change so the convention does not pull future shards toward the
  wrong domain/codomain.

## Cross-Shard Risks

- AQM-90 is careful about exact/proposal status for derivative predicates:
  it uses AQM-88's image definition and explicitly warns about fibre
  saturation. I did not find an unsupported claim that Kähler differentials
  directly produce toric-code boundaries.
- AQM-91 mostly preserves the coordinate-vs-translation separation required by
  convention `(az)`, but the exactness wording and the `F_p`/`F_2` transition
  are likely to propagate into later Haah/Laurent bridge shards if not fixed.
- No unsupported fracton or Haah-code construction claim appears in these two
  shards; the Haah material used here is toric-code/free-module grammar, not a
  new fracton construction.

## Suggested Actions

- Reword AQM-91's exactness paragraph so the equality is source-local over the
  infinite Laurent translation ring, while finite `R_G` carries a quotient to
  be measured.
- Specialize the AQM-91 toric-square bridge to qubit labels (`p=2`) before the
  identification with AQM-90, or explicitly mark the generic prime-qudit text
  as a separate generalization.
- Fix convention `(az)` in a later pass to distinguish the excitation map
  `sigma^dagger lambda` from the isotropy equation
  `sigma^dagger lambda sigma = 0`.
