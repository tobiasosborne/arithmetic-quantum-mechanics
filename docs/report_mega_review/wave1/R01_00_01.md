## Findings

- **Major:** `report/sections/00_frontmatter_status.tex:26-33` overstates the
  evidence-chain contract by saying every substantive assertion should be
  traceable through all listed items, including a `references/` source manifest
  and an `INDEX.md` row. This conflicts with the more flexible rule in
  `AGENTS.md:42-46`, where a report statement may be grounded by a local source,
  a checked local derivation, a test, or a reproducible run artifact. It also
  does not distinguish source-only claims, locally derived claims, open
  questions, and generated-data claims. As written, the frontmatter can make a
  purely local derivation look noncompliant for lacking an external source
  manifest, or make an open question look as though it needs the full
  source/run/index chain.

- **Major:** `report/sections/01_programme_map.tex:48-50` is stale in the
  current report. It says "The first technical session should not start with
  formulas" and should register sources/conventions first, but later included
  shards already record technical sessions and checked results, e.g.
  `report/sections/02_lab_log.tex:21-38` and `README.md:35-77`. In a current
  programme map, this reads as if no technical session has happened yet. It
  should either be explicitly historical or rewritten as a rule for the next
  unstarted bridge direction.

- **Minor:** `report/sections/00_frontmatter_status.tex:9-11` says the lab book
  tracks a programme "connecting" arithmetic zeta functions, arithmetic quantum
  mechanics, supersymmetric quantum mechanics, and topological lattice models.
  The local project rule is sharper: the connection is a research programme,
  not a settled claim (`AGENTS.md:15-17`), and the README uses "exploring a
  possible common language" (`README.md:3-6`). This opening sentence would be
  safer as "investigating possible bridges between..." or equivalent.

- **Minor:** `report/sections/01_programme_map.tex:19` asks which finite
  examples are small enough for exact Sage or GAP checks, but current tool
  conventions include Julia, Sage, GAP, and shell dispatch
  (`CONVENTIONS.md:84-97`), and current checked evidence includes Julia-backed
  runs (`report/sections/03_reproducibility_map.tex:23-33`). The question is
  not false, but it is now too narrow for the report's actual validation stack.

## Cross-Shard Risks

- The first two shards still read partly like the initial scaffold, while
  `AQM-02` onward records a much more developed checked layer. This can mislead
  readers about which parts are open programme setup versus completed local
  evidence.

- If `AQM-00` keeps the current "every assertion should be traceable through"
  wording, later reviewers may incorrectly flag valid local derivations or open
  questions for lacking source manifests or run-bundle/index links that are only
  conditionally required.

- The bridge-language risk is front-loaded: `AQM-00` appears before
  `AQM-01`'s explicit "map of questions" caveat, so imprecise wording there can
  color the whole report as claiming a connection rather than investigating one.

## Suggested Actions

- Rewrite `AQM-00`'s evidence-chain paragraph so it distinguishes alternatives:
  local source, convention entry when a convention is used, local derivation,
  test/run artifact for computation, `INDEX.md`/schema/run bundle only for
  generated-data claims, and explicit `Question`/`Proposal` status when evidence
  is intentionally missing.

- Rephrase `AQM-01`'s "First Decisions" section as current guidance for the
  remaining unstarted directions, or move it into a dated/historical scaffold
  note.

- Replace "connecting" in the `AQM-00` opening with "investigating possible
  bridges between" to match `AGENTS.md` and the README.

- Broaden the `AQM-01` exact-check question to "Julia/Sage/GAP or other
  registered local tools" while still requiring exact arithmetic or explicit
  tolerances where appropriate.
