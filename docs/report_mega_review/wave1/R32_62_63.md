## Findings

- Severity: Major
  Location: `report/sections/63_evaluated_derham_constraints_tangent_weyl.tex:51`
  Issue:
  The stabilizer phase statement is too broad: arbitrary eigenvalue phases do
  not automatically define a stabilizer subspace for a commuting Weyl label
  family.
  Evidence:
  Lines 47-52 correctly note that `0\oplus V_S^*` is isotropic and that the
  corresponding Weyl operators commute after central phases are fixed, but then
  say "Choosing eigenvalue phases then defines a stabilizer subspace." The
  project convention for phased stabilizers requires compatible phase data:
  `CONVENTIONS.md` lines 954-959 require a map satisfying
  `alpha(e+f)=alpha(e) alpha(f) c(e,f)^(-1)`. Without that compatibility, the
  joint eigenspace can be empty or the equations can fail to define a
  representation of the selected label subgroup.
  Suggested action:
  Replace the sentence with a compatibility-qualified version, e.g. "Choosing a
  compatible character/eigenvalue phase assignment, such as the all-`+1`
  assignment in the coordinate display below, defines the corresponding joint
  stabilizer eigenspace."

- Severity: Minor
  Location: `report/sections/63_evaluated_derham_constraints_tangent_weyl.tex:23`
  Issue:
  The definition of evaluated exact-one-form constraints omits the relative
  base data even though the construction uses `\Omega^1_{A/k}`.
  Evidence:
  Line 23 starts with `Let X=\Spec A`, while line 38 uses
  `\Omega^1_{A/k}`. Convention `(ao)` fixes this as relative cotangent data for
  a `k`-algebra (`CONVENTIONS.md` lines 1575-1587), and AQM-61 explicitly says
  `X=\Spec A` is presented by a `k`-algebra
  (`report/sections/61_tangent_cotangent_weyl_kinematics_examples.tex:28`).
  AQM-63 should make the same relative base explicit before evaluating `dA`.
  Suggested action:
  Start the definition with "Let `k` be the fixed base field, let `A` be a
  `k`-algebra, and let `X=\Spec A`." If later versions allow other bases,
  state which relative differential module is intended.

- Severity: Question
  Location: `report/sections/63_evaluated_derham_constraints_tangent_weyl.tex:29`
  Issue:
  The general finite-residue support notation may be overread as a vector-space
  direct sum over one field, while mixed-degree closed points only share an
  additive/`F_p` label structure.
  Evidence:
  Lines 23-31 allow a finite set of finite-residue points and define
  `V_S`, `V_S^*`, and `E_S` as direct sums. Lines 54-70 then correctly use the
  product of local characters `\psi_{K_P}` and separately specialize to the
  all-rational case. However AQM-62 explicitly includes degree-`d` closed
  points with `K=\mathbb F_{3^d}` at lines 48-50, so a support mixing rational
  and non-rational closed points is not naturally a `k`-vector-space support in
  the same sense as the examples. The text still calls the output a
  "subspace" at line 52 without saying whether this means a subgroup,
  `F_p`-subspace, `k`-subspace, or componentwise `K_P`-linear object.
  Suggested action:
  Add a sentence after the direct-sum definition: for mixed finite-residue
  supports these are additive, equivalently `F_p`-linear, Weyl label groups
  with local trace characters; in the all-`k`-rational examples they reduce to
  ordinary `k`-vector spaces. Then reserve `subspace` for the relevant
  category or spell out the intended one.

- Severity: Minor
  Location: `report/sections/62_tangent_cotangent_weyl_kinematics_more_examples.tex:13`
  Issue:
  The source anchors are indirect for a shard that introduces several new
  explicit tangent and carrier-dimension calculations.
  Evidence:
  Lines 13-14 say the shard "uses the same local sources" as AQM-61. The
  calculations appear consistent with AQM-61's presentation rule and with
  AQM-59's finite-field circle data, but the shard itself does not name the
  local Stacks files/line ranges or the specific prior report lines used for
  the tangent-presentation rule and finite Weyl carrier dimension. The project
  citation rule asks mathematical claims to point to a local path plus precise
  locator.
  Suggested action:
  Expand the status anchor by citing AQM-61's source-anchor lines for
  Kähler differentials/tangent spaces and the convention `(ao)` carrier
  dimension lines, and cite AQM-59 for the finite-field circle rational-point
  counts used in examples 9 and 10.

## Cross-Shard Risks

AQM-62 and AQM-63 mostly preserve the correction made in AQM-60--AQM-61:
tangent-cotangent Weyl data are kinematics, while stabilizers and dynamics
require an extra selection principle. The finite examples I checked are
consistent with that rule: affine-line and affine-plane carrier dimensions,
the crossing's `3^6` rational support, and the `F_3`/`F_5` circle counts match
the cited earlier computations.

The main downstream risk is terminology drift around "stabilizer subspace" and
"subspace" for evaluated de Rham constraints. If later shards cite AQM-63 for
mixed closed-point supports or for non-`+1` stabilizer phases, they need the
phase-compatibility and additive-label qualifications above.

## Suggested Actions

1. Qualify AQM-63's phase choice by the phased-stabilizer compatibility
   condition.
2. State the fixed relative base `k` in AQM-63 before using
   `\Omega^1_{A/k}` and `dA`.
3. Clarify mixed finite-residue support language as additive/`F_p`-linear
   unless all points are rational over the same base field.
4. Add precise local source/convention locators to AQM-62's status anchors.
