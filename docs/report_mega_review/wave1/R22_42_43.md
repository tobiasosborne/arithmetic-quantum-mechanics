## Findings

- Severity: Major
  Location: `report/sections/42_gaussian_clifford_dynamics.tex:237`
  Issue:
  The Gross-Hudson dictionary is stated as "pure Gaussian states are stabilizer
  states, equivalently phased Lagrangian label data" without separating
  `F_p`-additive stabilizers from `k`-linear Lagrangian label data when
  `k = F_{p^r}`.
  Evidence:
  Lines 25-28 correctly say the Gross-Hudson dictionary is imported for
  `k = F_{p^r}` only after choosing an `F_p`-basis and viewing
  `ell^2(k^m)` as `ell^2(F_p^{rm})`.  Gross's prime-power discussion gives the
  missing nuance: `references/symplectic_qecc/gross_0602001_source/poswig.tex:1800-1804`
  says the finite-field Wigner function agrees, after relabelling, with the
  `F_p^n` Wigner function, but lines 1819-1826 say the converse fails for
  stabilizer subspaces because an `F_p`-subspace need not be closed under
  `F_{p^n}`-scalar multiplication.  `CONVENTIONS.md:1154-1158` also says the
  `k`-linear dynamics is only a distinguished subclass of the `F_p`-linear
  Clifford dynamics.  The shard's "equivalently phased Lagrangian label data"
  can therefore be read as too strong unless it specifies which scalar field
  the Lagrangian is over.
  Suggested action:
  State the dictionary in two layers: Gross-Hudson gives pure nonnegative
  Wigner states as `F_p`-additive stabilizer/Lagrangian data after a basis
  choice; the report's intrinsic `k`-linear Lagrangians define a narrower
  finite-field Gaussian subclass unless an additional convention broadens
  "label data" to `F_p`-linear labels.  Add the local locator
  `poswig.tex:1703-1826`.

- Severity: Major
  Location: `report/sections/43_finite_field_scale_continuum_limits.tex:246`
  Issue:
  Frobenius is called a Clifford/Gaussian automorphism, but the displayed map
  is not `k_r`-linear and does not preserve the `k_r`-valued symplectic form in
  the sense fixed by AQM-42.
  Evidence:
  AQM-42 defines a symplectic morphism as a `k`-linear map preserving the
  `k`-valued form (`report/sections/42_gaussian_clifford_dynamics.tex:93-101`;
  `CONVENTIONS.md:1098-1127`).  In AQM-43,
  `F_r(q,p)=(q^3,p^3)` preserves the absolute trace character because
  `Tr(a^3)=Tr(a)` (`report/sections/43_finite_field_scale_continuum_limits.tex:232-248`),
  but over `k_r` it is Frobenius-semilinear, and
  `p^3q'^3-q^3p'^3 = (pq'-qp')^3`, not generally `pq'-qp'`.  The Stacks
  Frobenius source cited at lines 18-20 anchors the scheme Frobenius, not the
  Clifford/Gaussian classification under the AQM-42 `k`-linear convention.
  Suggested action:
  Rephrase T4 as an `F_3`-linear trace-Weyl automorphism, hence a Clifford
  automorphism after choosing the `F_3`-basis/Gross convention.  If "Gaussian"
  is retained, add a semilinear or `F_p`-linear Gaussian convention and say
  explicitly that this is not a `k_r`-linear symplectic automorphism.

- Severity: Major
  Location: `report/sections/43_finite_field_scale_continuum_limits.tex:125`
  Issue:
  The proof of the trace-normalized embedding checks the alternating
  commutator form, but the AQM-28 closed-point Weyl net uses a non-half Weyl
  multiplier.  A monomorphism of the written Weyl algebras needs preservation
  of the actual multiplier, not only the commutator.
  Evidence:
  AQM-28 defines `W_x(q,p)=T_x(q)R_x(p)` and proves
  `W(e)W(f)=psi(p q')W(e+f)` before deriving the commutator
  (`report/sections/28_closed_point_affine_arithmetic_fields.tex:114-130`).
  AQM-43 lines 125-138 compute preservation of
  `p q' - q p'` and then conclude at lines 146-153 that the Weyl relations give
  a `*`-monomorphism.  The multiplier preservation is true for the proposed
  map by the same trace argument applied to `p q'`, since only the momentum
  coordinate is scaled by `c_y`, but it is not the calculation currently shown.
  Suggested action:
  Add the multiplier check
  `sum_y Tr(c_y p_y q'_y) = Tr_K(p q')` before the commutator check, or
  explicitly switch this shard to the half-Weyl normalization of AQM-42 where
  preserving `1/2 Omega` is the multiplier statement.

- Severity: Minor
  Location: `report/sections/43_finite_field_scale_continuum_limits.tex:224`
  Issue:
  The explicit `F_{27}` trace-one example is asserted without a local source,
  derivation, or checked run.
  Evidence:
  Lines 224-227 say that if `theta^3 + 2 theta^2 + 1 = 0`, then this
  irreducible cubic has root trace `Tr(theta)=1`.  I found no local source or
  prior shard reference for this exact polynomial by text search.  The claim is
  easy to derive locally: the polynomial has no root in `F_3`, and for a monic
  cubic `x^3 + a x^2 + bx + c`, the trace of a root is `-a`, so here it is
  `-2 = 1` in `F_3`.  But that derivation is not recorded in the shard.
  Suggested action:
  Add the two-line finite-field derivation or cite a local checked artifact.

## Cross-Shard Risks

AQM-42 and AQM-43 are using two related but distinct layers of finite-field
Weyl structure: intrinsic `k`-linear symplectic maps and `F_p`-linear
trace-character Weyl maps after choosing a basis.  Gross-Hudson, Frobenius,
and full trace-gauge statements all become cleaner if downstream shards keep
those layers visibly separate.

The finite-field scale shards also rely on basic finite-field tower facts such
as `F_{q^r} subset F_{q^s}` iff `r | s`.  These facts are plausible and
standard, but the cited Stacks locators in AQM-43 mainly cover spectra,
residue fields, separable trace pairings, and Frobenius.  Downstream synthesis
should check whether a local finite-field source/derivation is intended for the
tower-lattice facts.

## Suggested Actions

1. Patch AQM-42 to distinguish `F_p`-additive Gross stabilizers from the

   locators added.
2. Patch AQM-43 Frobenius wording so it is an `F_p`-linear trace-Weyl/Clifford
   automorphism unless a semilinear Gaussian convention is added.
3. Add the missing Weyl multiplier preservation calculation for
   trace-normalized embeddings.
4. Add a local derivation for the `theta^3 + 2 theta^2 + 1` trace-one example
   or replace it with a cited/checked artifact.
