## Findings

- Severity: Major
  Location: `report/sections/54_derivative_dynamics_f3_examples.tex:107`
  Issue:
  The phrase "the remaining logical labels are controlled by the finite de Rham cohomology \(H^1_{\rm dR}(A/k)=\ker d_1/\operatorname{im}d_0\)" compresses two different CSS statements. The encoded qudit count is controlled by \(\dim H^1\), but the full logical Pauli label module is not just \(H^1\).
  Evidence:
  `CONVENTIONS.md:259`-`264` and `report/sections/08_symplectic_css_bridge.tex:184`-`195` identify the CSS logical Pauli module as \(L^\perp/L \cong H^1 \oplus H_1\). In the de Rham prototype, \(R_X=\operatorname{im}d_0\) and \(R_Z=\operatorname{im}d_1^T\), so the dual/homology side from the transposed complex is part of the logical Pauli labels even when the number of encoded qutrits is \(\dim H^1\).
  Suggested action:
  Replace "logical labels" with "encoded qutrit count/code dimension" there, or explicitly state that the full logical Pauli module is \(H^1\) plus the corresponding dual homology after the chosen basis pairing.

- Severity: Minor
  Location: `report/sections/55_derham_css_more_f3_examples.tex:250`
  Issue:
  The shard lists concrete stabilizer generators and then computes an all-\(+1\) stabilizer-space dimension, but it never states up front that the displayed generators use an added all-\(+1\) phase choice and that the ring calculation fixes only support spaces.
  Evidence:
  AQM-54 says the CSS test gives commuting stabilizer equations only "after choosing phases" at `report/sections/54_derivative_dynamics_f3_examples.tex:104`-`106` and says the test does not let the ring alone choose phases at lines `114`-`115`. AQM-55 lists `X(e_1)`, `X_1`, `X_3`, `X_2X_4`, and `Z_2^2Z_4` as selected stabilizers at `report/sections/55_derham_css_more_f3_examples.tex:161`-`167` and `250`-`256`, but does not include the phase caveat until the phrase "all-\(+1\)" inside the dimension sentence. The next shard states this cleanly at `report/sections/56_derham_css_f2_qubit_examples.tex:37`-`39`.
  Suggested action:
  Add a common-rule sentence in AQM-55 matching AQM-56: for a `C^1` basis vector \(e_i\), write \(X_i=X(e_i)\), \(Z_i=Z(e_i)\); all listed stabilizers use \(+1\) phase, while the de Rham calculation fixes only the support subspaces \(R_X,R_Z\).

- Severity: Question
  Location: `report/sections/55_derham_css_more_f3_examples.tex:277`
  Issue:
  The end summary says the prototype detects "nilpotent or singular first-order algebra", but the five examples only demonstrate zero-dimensional squarefree reduced cases with \(\Omega^1=0\) and nonreduced nilpotent cases with nonzero \(C^1\). A separate reduced singular example is not present in this shard.
  Evidence:
  The register table at `report/sections/55_derham_css_more_f3_examples.tex:38`-`49` classifies `E0,E1,E2` as reduced squarefree and `E3,E4` as nilpotent quadratic directions. The body computations then show no qutrits for `E0,E1,E2` at lines `50`-`129` and nilpotent-derived qutrits/stabilizers for `E3,E4` at lines `130`-`257`. No example isolates a reduced singular case or states the local theorem that, in these finite zero-dimensional perfect-field examples, singularity is necessarily tied to nilpotents.
  Suggested action:
  Narrow the sentence to "nilpotent first-order algebra in these examples" or add the missing local qualification explaining why "singular" is meant here and how it relates to nilpotents in the finite `F_3` Artin setting.

## Cross-Shard Risks

The main risk is CSS vocabulary drift. AQM-54 and AQM-55 correctly keep the de Rham CSS construction as a proposal/test rather than canonical tangent-Weyl dynamics, but "logical labels" and unqualified "stabilizer generators" can be read as stronger than convention `(ao)` permits. This matters for AQM-56 and AQM-57, which continue the same prototype and make code-parameter and entanglement claims from displayed stabilizers.

## Suggested Actions

1. Patch AQM-54's logical-label sentence to distinguish encoded-count control by \(H^1_{\rm dR}\) from the full CSS logical Pauli module.
2. Add an explicit all-\(+1\) phase/support-space caveat near the start of AQM-55 before the stabilizer lists.
3. Narrow or justify the "nilpotent or singular" end-summary wording in AQM-55.
