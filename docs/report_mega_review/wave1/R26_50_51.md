## Findings

- Severity: Minor
  Location: `report/sections/50_f3_regular_functions_line_by_line.tex:48`
  Issue:
  The shard asserts the complete list of monic irreducible quadratics over
  `F_3` without a local derivation or source locator.
  Evidence:
  Lines 48-50 say "The three monic irreducible quadratics are" and list
  `x^2+1`, `x^2+x+2`, and `x^2+2x+2`. The list is mathematically correct, but
  the shard does not show the finite enumeration/root check or cite a local
  source/run for completeness. The same section's source anchors at lines
  11-14 cite broad shard dependencies, not a locator for this explicit
  enumeration.
  Suggested action:
  Add a one-sentence derivation: over `F_3`, a monic quadratic is reducible iff
  it has an `F_3` root; checking the nine monic quadratics leaves exactly these
  three. Alternatively cite a prior local line that already performs this
  enumeration.

- Severity: Minor
  Location: `report/sections/50_f3_regular_functions_line_by_line.tex:157`
  Issue:
  The notation `q_0` is reused for two different objects in the same shard.
  Evidence:
  Lines 20-21 and 29 define `q_0=x^2+1` as the chosen irreducible quadratic,
  and later lines 197-223 use `q_0` again as that support polynomial. But lines
  157-159 prescribe a two-qutrit label with `q_0=2, q_1=1`, where `q_0,q_1`
  now mean position labels at the rational sites. This is easy to confuse with
  the polynomial `q_0` and with the quadratic list `q_0,q_1,q_2`.
  Suggested action:
  Rename the two-qutrit label coordinates to something like
  `Q_0=2, Q_1=1` and `P_0=1, P_1=0`, reserving `q_i` for quadratic
  polynomials in this shard.

- Severity: Question
  Location: `report/sections/51_regular_function_affine_symmetry_action.tex:230`
  Issue:
  The thickened transport paragraph correctly warns that reduced covariance
  does not prove covariance for independently chosen Artin characters, but the
  positive covariance statement is still terse enough to be overread as a
  default-character result.
  Evidence:
  Lines 230-235 say a thickened Weyl automorphism needs a character statement:
  transported target characters give covariance, while separately chosen
  default top-coefficient characters require a dual momentum correction. This
  matches convention `(al)` in `CONVENTIONS.md:1503-1514`. However AQM-36 fixes
  the default character by the top `pi`-coefficient
  (`report/sections/36_nilpotent_thickening_weyl_fields.tex:84-90`), and
  AQM-51 does not display how that top-coefficient functional changes under
  `sigma_g(pi)=unit * pi^g`. A downstream reader could mistakenly apply T4 as
  Weyl covariance for the default source and default target characters without
  the promised correction.
  Suggested action:
  Keep T4 as a ring/support transport theorem, and add an explicit sentence
  before the covariance claim: "No default top-coefficient Artin character is
  being claimed invariant here." If useful, defer the actual correction formula
  to the line-by-line F3 companion or a later thickened-symmetry shard.

## Cross-Shard Risks

AQM-50 and AQM-51 are consistent with the main regular-function convention:
nonzero regular functions are infinite residue profiles, while finite Weyl
operators appear only after choosing a finite support selector or a finite
Artin thickening. The main risk is terminology and notation drift, not a false
dimension or covariance calculation.

The Artin transport caveat in AQM-51 lines 230-235 is important because AQM-52
uses concrete transformed repeated-factor supports. Downstream shards should
not cite reduced affine covariance as evidence for thickened Weyl covariance
unless the generating character is transported or the momentum-label correction
is explicitly supplied.
