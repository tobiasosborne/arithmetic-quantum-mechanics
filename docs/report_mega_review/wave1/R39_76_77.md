## Findings

- Severity: Question
  Location: `report/sections/76_quantum_system_association_cotangent_first.tex:34`
  Issue:
  AQM-76 opens the cotangent-first workflow for a scheme over a "named base
  field or base ring", but the displayed recipe immediately writes
  `T^*_{X/k,x} = \Omega^1_{A/k}\otimes_A K_x` at lines 46--49. This is safe for
  the field-base anchors AQM-58--AQM-63, but it is notation-sensitive for the
  base-ring cases that AQM-77 then reviews.
  Evidence:
  AQM-77 lines 50--62 define the base-dependent object
  `C_x^{rel}(B)=\Omega^1_{A/B}\otimes_A K` and record the local quotient that
  removes base directions. AQM-77 lines 210--229 show the sharp case
  `A=Z/4Z`: intrinsic `m/m^2` gives a qubit-sized cotangent-first carrier, while
  relative over `Z` gives `\Omega^1_{A/Z}=0` and a one-dimensional carrier.
  Convention `(av)` at `CONVENTIONS.md:2168`--`2173` requires naming the
  cotangent object before quoting phase-label dimensions or Weyl carrier sizes.
  Suggested action:
  Either restrict the AQM-76 displayed workflow explicitly to a field base `k`
  and leave base-ring notation to AQM-77, or rewrite the generic recipe with a
  named base `B` as `T^*_{X/B,x}` / `\Omega^1_{A/B}` and state that the
  intrinsic `m_x/m_x^2` alternative is introduced only in AQM-77.

## Cross-Shard Risks

AQM-77 correctly prevents the main downstream overclaim: it distinguishes
intrinsic/local cotangent data from relative Kähler cotangent data, gives
agreement examples over `F_3`, gives a base-change disagreement example over
`k[u]/(u^2)`, and gives the relative-over-`Z` collapse for `Z/4Z`. The remaining
risk is that downstream shards might cite the shorter AQM-76 recipe and forget
the base choice, especially for finite rings over `Z` or over a non-field
finite base.

The finite-ring database caveat is handled cleanly in both assigned shards:
AQM-76 lines 223--227 and AQM-77 lines 250--253 do not claim populated
database-backed cotangent rows, helper coverage, or finite-ring-wide tables.
This matches `INDEX.md` and the finite-ring run README, which say the SQLite
database is schema-only/local and the CSVs are in-memory MVP review exports
rather than an audited populated catalogue.

## Suggested Actions

- Make the AQM-76 base notation match the later AQM-77 distinction before later
  QSA shards reuse the cotangent-first recipe for finite rings.
- Keep AQM-77's intrinsic-versus-relative examples and finite-ring database
  caveats unchanged for Wave 1 purposes.
