## Findings

- Severity: Minor
  Location: `report/sections/81_quantum_system_association_residue_field_site_association.tex:153`
  Issue:
  The sourced example table introduces nonstandard status-marker forms
  `\texttt{blocked/database}`, `\texttt{not_claimed}`, and
  `\texttt{available locally}`.
  Evidence:
  Convention `(au)` defines the report-planning status tokens as lowercase
  labels such as `available`, `blocked`, `unknown`, `not_applicable`,
  `degenerate`, `zero`, and `empty`, and says the tokens are distinct from CSV
  sentinel values. `AQM-80` follows this discipline explicitly at lines 40--50.
  In `AQM-81`, the `\mathbb Z/4\mathbb Z` row uses
  `\texttt{blocked/database}` at line 153 and `\texttt{not_claimed}` at line
  156; the dual-number row uses `\texttt{available locally}` at line 162. These
  are understandable prose markers, but they are easy to misread as additional
  status vocabulary or schema-local finite-ring statuses.
  Suggested action:
  Rewrite these as standard tokens plus prose, e.g. `\texttt{blocked}: database
  residue helper does not certify the row; AQM-77 supplies only a local
  cotangent anchor`, `No Step 17 database-backed residue-site row is claimed`,
  and `\texttt{available}: local AQM-34/AQM-80 anchor only; database residue
  helper remains \texttt{blocked}`.

## Cross-Shard Risks

The main cross-shard risk is status drift between report-planning tokens,
finite-ring CSV statuses, and prose blockers. `AQM-80` handles this carefully:
it states that the 2026-05-26 SQLite artifact is schema-only, that the CSVs are
in-memory MVP review exports, and that `Z/4Z` and dual-number database residue
rows remain blocked. `AQM-81` makes the same mathematical/database distinction,
but the ad hoc marker spellings above weaken the visual discipline.

I did not find a populated finite-ring database overclaim in these two shards.
The statements about `Z/4Z`, dual numbers, mixed residues for `Z/6Z`,
cotangent-only local anchors, and nilpotent-sensitive separation match the
local conventions and the finite-ring run README/export status I inspected.

## Suggested Actions

- Normalize `AQM-81` table markers to the convention `(au)` token vocabulary
  plus explanatory prose.
- Keep the current explicit distinction between local hand-derived anchors and
  blocked database-backed rows; it is doing useful work in both shards.
