## Findings

- Severity: Major
  Location: `report/sections/12_steane_clifford_koszul_morphisms.tex:103`
  Issue:
  The transversal phase subsection correctly says it is ignoring Pauli phases
  in the binary label calculation, but it never restores the signed/logical
  caveat before discussing logical labels. This can make the bitwise phase look
  like an encoded phase with logical action
  `Xbar -> Xbar + Zbar, Zbar -> Zbar`, while the registered Gottesman source
  says bitwise `P` on the seven-qubit code converts `Xbar` into `-Ybar` and
  performs encoded `P^\dagger`.
  Evidence:
  `report/sections/12_steane_clifford_koszul_morphisms.tex:105-143` works only
  with unsigned binary labels for `P^{\otimes 7}`. The cited local source
  `references/symplectic_qecc/gottesman_9705052_source/Thesis.tex:2439-2445`
  records the sign-sensitive encoded action and explicitly identifies it as
  `P^\dagger`. `AQM-13` later explains why projectors and signed Paulis cannot
  silently drop phases (`report/sections/13_clifford_group_ghost_gaussian_theorem.tex:19-44`).
  Suggested action:
  Add a sentence or small displayed calculation in `AQM-12` separating the
  unsigned symplectic-label action from the signed encoded operation, and cite
  the Gottesman lines that identify the operation as encoded `P^\dagger`. The
  run/schema columns for `phase_logical_x_image` and `phase_logical_z_image`
  should also be described as binary-label images, not signed logical action.

- Severity: Major
  Location: `report/sections/12_steane_clifford_koszul_morphisms.tex:156`
  Issue:
  The "Exact Validation" paragraph overstates the independent evidence supplied
  by the run bundle for the chain-map and homotopy-equivalence claims. The
  report says the producer records the Hadamard/phase chain-map data, the 63
  contractible nonzero syndrome sectors, and cohomology dimensions, but the
  inspected producer/library path mostly emits summary booleans and strings for
  these claims rather than a local certificate containing the actual chain maps,
  contractions, or matrix residuals.
  Evidence:
  The CSV `runs/2026-05-24-steane-clifford-koszul-morphisms/data/steane_clifford_koszul_summary.csv`
  has booleans such as `phase_chain_isomorphism_to_image_presentation=true`,
  `phase_same_q_only_after_homotopy_retract=true`, and
  `nonzero_syndrome_blocks_contractible=true`. The producing function
  `src/ArithmeticQuantumMechanics/SteaneSupercharge.jl` hard-codes several of
  these fields, including `phase_same_q_only_after_homotopy_retract = true`,
  `nonzero_syndrome_blocks = 63`, and
  `nonzero_syndrome_blocks_contractible = true`; the tests in
  `test/runtests.jl` assert those summary fields. The mathematical argument in
  `AQM-11` supports acyclicity abstractly, but the run artifact is not itself a
  detailed exact chain-homotopy certificate.
  Suggested action:
  Either soften `AQM-12:156-162` to say the producer records a summary
  certificate consistent with the local derivation, or extend the artifact so it
  writes/checks explicit chain-map identities and contraction data with a schema
  entry that makes the certificate contract clear.

## Cross-Shard Risks

- `AQM-12` intentionally uses unsigned binary labels for the transversal phase,
  while `AQM-13` correctly warns that signed projectors are the general
  convention. Without a back-reference or signed correction in `AQM-12`, the
  two shards can look inconsistent about when Pauli phases matter.

- The report's "exact validation" wording is stronger than the artifact format
  in at least this local run. Later shards may inherit the same pattern of a
  summary CSV being described as a full certificate unless the certificate
  contract is made explicit.

- The ghost-Gaussian theorem in `AQM-13` is internally careful about
  operator-valued coefficients and no Bogoliubov mixing. I did not find a local
  contradiction there, but it depends on readers not importing the scalar
  `GL_r(F_2)` intuition back into the full physical Hilbert-space chain.

## Suggested Actions

- Patch `AQM-12` so the transversal phase subsection explicitly distinguishes
  binary symplectic-label action from signed encoded Clifford action, including
  the Gottesman `P^\dagger` locator.

- Reword the `AQM-12` exact-validation paragraph or strengthen the run artifact
  so "exact validation" means an inspectable chain-map/homotopy certificate,
  not only summary booleans.

- Fix the shortened producer and source paths in `AQM-12`.
