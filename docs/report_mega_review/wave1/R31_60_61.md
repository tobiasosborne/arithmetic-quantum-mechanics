## Findings

- Severity: Major
  Location: `report/sections/60_tangent_weyl_operator_principle.tex:31`
  Issue:
  The general Lamport skeleton and final workflow omit the base field/base map
  needed to define the relative cotangent object before using
  `\Omega^1_{A/k}`, tangent spaces, carrier dimensions, and residue-Weyl
  comparisons.
  Evidence:
  AQM-60 starts with `X=\Spec A, P\in X, K=\kappa(P)` at lines 31--34 and ends
  with "Choose `X=\Spec A`" at lines 208--215, but the cotangent formula uses
  `\Omega^1_{A/k}`. `CONVENTIONS.md` convention `(ao)` defines the tangent
  recipe for a commutative `k`-algebra and convention `(av)` explicitly says a
  shard must state which cotangent object it uses before quoting phase-label
  dimensions or carrier sizes (`CONVENTIONS.md:1575` and
  `CONVENTIONS.md:2168`). AQM-61 does name `k` in its recipe at
  `report/sections/61_tangent_cotangent_weyl_kinematics_examples.tex:28`, so
  the gap is specifically in AQM-60's advertised principle.
  Suggested action:
  Add the base data to AQM-60 D1 and the operational workflow, e.g.
  `k -> A` or `A` a `k`-algebra, and state that all displayed tangent/cotangent
  spaces in the shard are relative to that base. For the real example, name the
  base as `\mathbb R`.

- Severity: Major
  Location: `report/sections/60_tangent_weyl_operator_principle.tex:42`
  Issue:
  The finite-support isotropy step is under-specified for supports with mixed
  residue fields. It says `E_S=\oplus E_P` and asks for vanishing symplectic
  pairings, but the project already records that mixed-residue finite supports
  do not have one field-valued symplectic form.
  Evidence:
  AQM-60 lines 42--47 and 90--98 state the direct-sum support and isotropic
  stabilizer step using "symplectic pairings vanish." AQM-61 repeats the
  finite-support tensor product at lines 69--74 without restating the mixed-field
  commutator convention. By contrast, AQM-28 says that for closed finite
  supports the form is "not a form into one field when the residue degrees vary"
  and the representation uses the alternating bicharacter
  `B_S(e,f)=\prod_x \psi_x(...)`
  (`report/sections/28_closed_point_affine_arithmetic_fields.tex:93`).
  This matters because the surrounding tangent-Weyl programme includes closed
  points of higher degree, not only rational `F_3` supports.
  Suggested action:
  Either restrict these two shards' stabilizer-selection language to a common
  residue field/rational support, or define isotropy for finite supports as
  triviality of the character-valued Weyl commutator bicharacter, with the
  local trace characters included.

- Severity: Minor
  Location: `report/sections/60_tangent_weyl_operator_principle.tex:74`
  Issue:
  The finite tangent-cotangent Weyl representation dimension is stated as
  "the irreducible finite representation" with only AQM-28 as the local anchor,
  but AQM-28 displays the one-site residue label `\kappa(x)^2` case rather than
  the general `Q\oplus Q^*` finite-dimensional tangent carrier.
  Evidence:
  AQM-60 lines 74--83 and AQM-61 lines 51--67 use the general carrier formula
  `|\kappa(P)|^{\dim T_PX}`. The formula is consistent with
  `CONVENTIONS.md:1632` and with the later QSA synthesis at
  `report/sections/78_quantum_system_association_finite_phase_weyl_kinematics.tex:30`,
  but AQM-60's source-anchor paragraph points only to AQM-28 for finite Weyl
  operators (`report/sections/60_tangent_weyl_operator_principle.tex:19`).
  Suggested action:
  Add a local anchor to convention `(ao)` or to the finite phase-space display
  in AQM-78/AQM-76, or include the short tensor-product derivation from
  `\ell^2(T_PX)` directly in AQM-60/AQM-61.

## Cross-Shard Risks

AQM-63 uses the same shorthand `T_PX`, `T_P^*X`, and `E_S` in its definition of
evaluated de Rham constraints. If AQM-60 is clarified to require a named base
and a character-valued mixed-residue isotropy condition, AQM-63 should be swept
for the same wording.

The residue-Weyl versus tangent-Weyl separation is otherwise clearly maintained
in these two shards: product fields are zero in the tangent-cotangent layer,
the `F_3` nilpotent examples are presented as kinematics, and no Hamiltonian or
dynamics is claimed.

## Suggested Actions

1. Patch AQM-60's D1 and final workflow to include the base for
   `\Omega^1_{A/k}`.
2. Define finite-support isotropy using the Weyl commutator bicharacter, or
   explicitly restrict the displayed isotropic-subspace language to common-field
   supports.
3. Add a precise local anchor for the general finite `Q\oplus Q^*`
   Weyl carrier dimension used by the tangent-cotangent recipe.
