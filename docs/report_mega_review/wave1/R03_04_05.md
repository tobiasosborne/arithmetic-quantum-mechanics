## Findings

- **Major** - `report/sections/05_toric_supercharge_checks.tex:201-202` says logical string operators are labelled by the paired `\(H_1/H^1\)` data. This is not a defined quotient in the local development and conflicts with the later symplectic-CSS statement `L^\perp/L\cong H^1\oplus H_1` in `report/sections/08_symplectic_css_bridge.tex:184-195` and the complementary logical Lagrangians in `report/sections/08_symplectic_css_bridge.tex:208-226`. In AQM-05 itself, the code basis is labelled by `H^1` at `report/sections/05_toric_supercharge_checks.tex:197-201`, while AQM-06 identifies the code space with `\mathbb C[H^1_{\mathrm{cell}}]` at `report/sections/06_toric_chain_ghost_proof.tex:117-138`. Suggested correction: replace `\(H_1/H^1\)` with language like "paired `H^1` and `H_1` data" or defer the logical Pauli module statement to AQM-08.

- **Minor** - `report/sections/04_source_queue_open_questions.tex:11-18` makes a broad source-coverage claim without local manifest pointers. `INDEX.md:193-208` does list the relevant source-topic manifests, but AQM-04 itself does not point readers to those rows or to `references/**/SOURCES.md`. This is not a mathematical error, but under the repo citation discipline the source queue should make the provenance path explicit.

- **Minor** - `report/sections/05_toric_supercharge_checks.tex:67-71` displays `QQ^*+Q^*Q=\sum_iP_i=H_{\mathrm{TC}}` before saying in prose that the full-space equality is `H_{\mathrm{TC}}\otimes I`. This is easy to misread as an equality of operators on the ghost-extended space without the ghost identity. `CONVENTIONS.md` convention `(j)` states the full-space comparison as `{Q,Q^*}=H_TC tensor I_ghost`. Make the displayed equation explicit and reserve `H_{\mathrm{TC}}` alone for the ghost-vacuum restriction.

- **Minor** - `report/sections/05_toric_supercharge_checks.tex:100-104` says "Since the `P_i` are commuting orthogonal projectors". They are self-adjoint orthogonal projections and they commute, but they are not pairwise orthogonal projectors; simultaneous violated checks can coexist. The quotient statement is salvageable via the simultaneous decomposition for commuting projections, but the wording should be "commuting self-adjoint projectors" or "commuting orthogonal projections" to avoid implying `P_iP_j=0` for `i\ne j`.

- **Question** - `report/sections/05_toric_supercharge_checks.tex:108-121` says the validation script verifies the CAR/projector certificates for `Q^2=0` and the anticommutator. The CSV sentinel in `runs/2026-05-24-toric-supercharge/data/toric_supercharge_summary.csv:1` correctly says this is algebraic only, and the implementation records `q_square_certificate` and `anticommutator_certificate` as boolean consequences of commuting/squaring checks in `src/ArithmeticQuantumMechanics/ToricCodeSupercharge.jl:179-180`. That may be acceptable because AQM-05 sketches the proof at `report/sections/05_toric_supercharge_checks.tex:62-68` and AQM-06 proves it at `report/sections/06_toric_chain_ghost_proof.tex:186-204`, but the run-evidence language would be less overclaiming if it said the run records the checked hypotheses used by the CAR/projector proof.

- **Minor** - `report/sections/05_toric_supercharge_checks.tex:218-219` gives the second CSV as `toric_chain_ghost_unification.csv` in the `2026-05-24-toric-chain-ghost-unification` run bundle, omitting both `runs/` and `data/`. `INDEX.md:61` and `data/SCHEMA.md:20-21` use the full path `runs/2026-05-24-toric-chain-ghost-unification/data/toric_chain_ghost_unification.csv`. Use the full path in the shard for traceability.

## Cross-Shard Risks

- AQM-05 is an introductory shard but already states logical-label structure. The `H_1/H^1` wording risks propagating a wrong quotient before AQM-08 gives the correct `H^1\oplus H_1` logical Pauli module.
- AQM-05's run-evidence paragraph depends on the detailed proof in AQM-06 and the operator distinction in AQM-07. The shard mostly flags this, but the verification wording should stay aligned with the CSV sentinel: algebraic certificate, no full Hilbert or ghost matrices.
- AQM-04's source queue is high-level and currently relies on external navigation through `INDEX.md`; if source status changes, this shard can become stale unless it cites the manifest rows it summarizes.

## Suggested Actions

- Replace `\(H_1/H^1\)` in AQM-05 with the correct paired/direct-sum logical-label language and cross-reference AQM-08 if needed.
- Tighten AQM-05's anticommutator display to include `\(\otimes I_{\mathrm{ghost}}\)` on the full ghost space.
- Reword the degree-zero homology paragraph to avoid implying pairwise-orthogonal projectors.
- Change the run-evidence wording from "verifies/certifies" to "records the checked hypotheses used by the CAR/projector proof", or cite AQM-06 as the proof of the certificate.
- Add manifest or `INDEX.md` pointers to AQM-04's broad registered-source summary and use the full CSV path for the chain-ghost run in AQM-05.
