- Severity: Major
  Location: `report/sections/79_quantum_system_association_geometric_field_hom_x_v.tex:185`
  Issue:
  The regular-function branch blurs two different uses of a polynomial `h`: denominator/open data for `D(h)` and closed-support selector data for `Z_{\rm cl}(h)`. This can make `W_h^{\rm red}(F,G)` look like it is defined for the localized regular functions just introduced on `D(h)`.
  Evidence:
  Lines 185--190 introduce `(F,G)\in \mathbb F_q[x]_h^2` on `U=D(h)`, where residue evaluations are only at closed points with `\pi\nmid h`. Lines 196--206 then use the same displayed `h` to define `Z_{\rm cl}(h)=\{\pi:\pi\mid h\}` and `W_h^{\rm red}(F,G)` by reductions modulo those `\pi`. But AQM-49 separates these roles: `report/sections/49_regular_functions_affine_line_weyl_labels.tex:64` defines localized evaluations on `D(h)` only for `\pi\nmid h`, while `report/sections/49_regular_functions_affine_line_weyl_labels.tex:144` defines `W_h^{\rm red}` for polynomial pairs `(F,G)\in R^2` reduced at `\pi\mid h`. A general element of `R_h` has denominators that are not evaluable at the support points `\pi\mid h`.
  Suggested action:
  Split this paragraph into two explicitly different variables or roles, e.g. `U=D(s)` with `(F,G)\in R_s^2` for infinite residue profiles, and a separate nonzero support selector `h` with polynomial labels `(F,G)\in R^2` or classes modulo `\operatorname{rad}(h)` for `W_h^{\rm red}`. Also tighten the choice-table row at lines 86--93 so it does not suggest that the same localized regular-function datum directly produces the reduced finite-support operator.
