## Findings

- Severity: Major
  Location: `report/sections/44_trace_gauges_full_continuum_towers.tex:136`
  Issue:
  The full-tower Weyl-net theorem is stated for an arbitrary affine `k`-scheme `X_0=\Spec R`, but the closed-point Weyl-net machinery it invokes is only justified for finite-residue closed points, locally fixed in convention `(y)` for affine schemes of finite type over `F_q`.
  Evidence:
  AQM-44 starts T2 with only "Let `X_0=\Spec R` be affine over `k`" at lines 136-140, then asserts monomorphisms of all open-local Weyl nets at lines 147-164. `CONVENTIONS.md` lines 748-758 restrict the closed-point finite-residue convention to affine schemes of finite type over `F_q`, because closed residues are then finite. Without that hypothesis, a closed point can have infinite residue field, and the finite Weyl/qudit site is not defined by AQM-28.
  Suggested action:
  Add a finite-type-over-`F_q` hypothesis, or explicitly state "for those closed points with finite residue fields" and prove the net restriction under that hypothesis. Mirror the same hypothesis in any convention or summary that says the construction works for an arbitrary affine base.

- Severity: Minor
  Location: `report/sections/44_trace_gauges_full_continuum_towers.tex:87`
  Issue:
  The factorial-tower construction uses trace transitivity as a ground-truth step, but the status anchors cite only separable trace-pairing nondegeneracy and Frobenius source lines.
  Evidence:
  The independence and coherence proof uses identities of the form `Tr_{N_n/r} = Tr_{N_m/r}(Tr_{N_n/N_m}(...))` at lines 87-105. The source anchors at lines 17-21 cite Stacks lines for nondegenerate separable trace pairing and Frobenius, not trace transitivity. Under the repository law, this should be either locally derived from linear-algebra trace of multiplication maps or source-anchored.
  Suggested action:
  Add a short derivation of trace transitivity for finite field extensions, or cite a local source line that states it.

- Severity: Major
  Location: `report/sections/45_spec_z_arithmetic_quantum_field.tex:130`
  Issue:
  The closed-prime net proof says "This is AQM-28 applied", but AQM-28 is not literally applicable to `Spec(Z)`.
  Evidence:
  AQM-45 applies AQM-28 at lines 129-138. AQM-28 and `CONVENTIONS.md` `(y)` are for finite-type affine schemes over finite fields; `Spec(Z)` is not over `F_q`. AQM-45 does define the finite prime residues directly at lines 83-117, so the construction is probably valid, but the provenance statement overstates what AQM-28 supplies.
  Suggested action:
  Rephrase as "the same finite-support construction as AQM-28, using the finite closed residues proved above" and keep the direct proof. If this is intended as a new general convention, state the exact extension in `(ai)`: selected closed points with finite residue fields, not an automatic application of `(y)`.

- Severity: Major
  Location: `report/sections/45_spec_z_arithmetic_quantum_field.tex:180`
  Issue:
  The optional generic `Q` sector uses `\mathcal A_\eta` in the local rule without defining that algebra or its completion status.
  Evidence:
  Lines 161-176 define `E_eta=Q^2`, `H_eta=ell^2(Q)`, `chi_Q`, and the translation/modulation operators, but line 182 uses `\mathcal A_\eta\otimes\mathcal A_{\rm cl}(U)` without defining whether `\mathcal A_\eta` is the algebraic span of Weyl operators, the generated bounded-operator algebra, a group algebra quotient, or some completion. This matters because nearby source-status shards warn that countably infinite discrete fields/rings are not automatically self-dual LCA Weyl systems.
  Suggested action:
  Define `\mathcal A_\eta` explicitly as an algebraic object and state that no Stone-von Neumann uniqueness, `C^*`-completion, adelic replacement, or full Pontryagin-dual phase space is being claimed. If the intended object is only a proposal, mark it as such in the theorem skeleton.

- Severity: Major
  Location: `report/sections/45_spec_z_arithmetic_quantum_field.tex:267`
  Issue:
  The conclusion says the infinite algebra sees "the profinite completion", but the constructed action only sees the squarefree residue quotient `prod_l F_l`.
  Evidence:
  T3 proves finite support periods `N_S=prod_{\ell in S}\ell` at lines 225-240 and extends the action to `prod_l F_l`, then to `\widehat Z` only after coordinatewise reduction modulo `p` at lines 229-231. The map `\widehat Z=prod_p Z_p -> prod_p F_p` has nontrivial kernel `prod_p p Z_p`, so the action is not faithful on the full profinite completion. Line 267's "the profinite completion" therefore overstates the result.
  Suggested action:
  Replace the conclusion with "the squarefree profinite residue quotient `prod_l F_l`" or "a quotient of `\widehat Z` via reduction modulo each prime." State explicitly that the extended `\widehat Z`-action factors through this quotient.

- Severity: Minor
  Location: `report/sections/45_spec_z_arithmetic_quantum_field.tex:194`
  Issue:
  The no-scheme-translation theorem relies on affine scheme anti-equivalence, but the source anchor lists only `stacks_project_algebra.tex` lines.
  Evidence:
  Lines 193-198 use the correspondence between scheme endomorphisms of `Spec Z` and unital ring endomorphisms of `Z`. The local source manifest records affine scheme morphisms and anti-equivalence in `references/algebraic_geometry/stacks_project_schemes.tex`, lines 841-963, while AQM-45's status anchor at lines 12-18 cites only `stacks_project_algebra.tex` for "functoriality."
  Suggested action:
  Add the `stacks_project_schemes.tex` citation to the status anchors for T2, or derive the affine endomorphism correspondence locally before using it.

## Cross-Shard Risks

AQM-43 and convention `(ag)` use the same broad "affine `k`-scheme" language for finite-field scale systems. If AQM-44 is tightened to finite type or finite-residue supports, AQM-43 and `CONVENTIONS.md` lines 1162-1220 should be checked for the same hypothesis drift.

AQM-84 correctly says the `Spec(Z)` incomplete tensor representation is not a `C^*`-completion and that profinite/adelic sectors need their own conventions. AQM-45 should stay aligned with that caution by avoiding "the profinite completion" when only the squarefree residue quotient is constructed.

## Suggested Actions

1. Tighten AQM-44 T2 to finite-type-over-finite-field or finite-residue-support hypotheses.
2. Add a local source or derivation for trace transitivity in AQM-44.
3. Reword AQM-45's use of AQM-28 as an analogy/extension, not a literal application.
4. Define `\mathcal A_\eta` and mark the generic `Q` sector's completion status.
5. Replace "the profinite completion" with the squarefree quotient `\prod_\ell F_\ell`, with any `\widehat Z` action described as factoring through reduction.
6. Add the Stacks `schemes.tex` anti-equivalence source anchor for the `Spec(Z)` endomorphism theorem.
